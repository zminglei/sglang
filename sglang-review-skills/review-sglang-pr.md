# /review-sglang-pr — SGLang PR Expert Review

## Invocation

The user provides a PR reference (URL or number). Default repo is `sgl-project/sglang`.

## Step-by-Step Procedure

### Step 1: Fetch PR Data

```bash
# Parse the PR reference
# If just a number, use sgl-project/sglang
gh pr view <PR_NUMBER> --repo sgl-project/sglang --json title,body,files,number,url
gh pr diff <PR_NUMBER> --repo sgl-project/sglang
```

Extract:
- `title`: PR title
- `body`: PR description
- `files`: List of changed file paths (use `.path` from the JSON)
- `diff`: Full diff text

### Step 2: Route to Domain Experts

Use the router in `sglang-review-skills/router.py`:

```python
import sys
sys.path.insert(0, "sglang-review-skills")
from router import route_pr, format_routing_summary

# changed_files = list of file paths from step 1
# diff_content = the diff text from step 1
routes = route_pr(changed_files, diff_content)
routing_summary = format_routing_summary(routes)
```

Show the routing summary to the user. The top experts (confidence > 0.3) will review the PR.

### Step 3: Expert Reviews

For each selected expert, launch a **sub-agent** (using the Agent tool) that:

1. Reads the expert prompt from `sglang-review-skills/experts/<expert_name>.md`
2. Reads the PR diff
3. For deeper context, reads the actual changed files from the SGLang repo on the pod:
   ```bash
   kubectl exec f04099ace82404b8dbe6-n0-0 -n kingkong-dev --context prod-ltx1-k8s-2 -- \
     cat /home/jobuser/zminglei/sglang/<file_path>
   ```
4. Produces a structured review following the expert's prompt format

**IMPORTANT**: Launch expert agents **in parallel** for efficiency. Each expert is independent.

The expert agent prompt should be:
```
You are an SGLang code reviewer specializing in {expert_description}.

{expert_prompt_content}

Review this PR:
- Title: {pr_title}
- Description: {pr_body}
- Changed files: {files_list}

Diff:
{diff}

For any files you need deeper context on, read them from the SGLang repo:
kubectl exec f04099ace82404b8dbe6-n0-0 -n kingkong-dev --context prod-ltx1-k8s-2 -- \
  cat /home/jobuser/zminglei/sglang/{file_path}

Provide your review in this format:
### {expert_name} Review
**Risk Level**: [Low / Medium / High / Critical]
**Summary**: 1-2 sentences
**Issues Found**: (bulleted list, or "None")
**Suggestions**: (bulleted list, or "None")
**Looks Good**: (what's well done, or skip if nothing notable)
```

### Step 4: Aggregate Reviews

Combine all expert reviews into a single comment:

```markdown
# 🔍 SGLang Domain Expert Review

**PR**: {title} (#{number})

## Routing
{routing_summary}

---

{expert_1_review}

---

{expert_2_review}

---

...

*Generated by SGLang domain expert review agents.*
```

### Step 5: Post Review (with confirmation)

**Ask the user** before posting:
- Show them the aggregated review
- Ask: "Post this as a comment on the PR?"

If confirmed:
```bash
gh pr comment <PR_NUMBER> --repo sgl-project/sglang --body "$(cat <<'EOF'
{aggregated_review}
EOF
)"
```

## Notes

- Maximum 4-5 experts per PR to keep reviews focused
- If a PR touches only 1-2 domains clearly, skip routing and go straight to those experts
- For very large PRs (>1000 lines), focus experts on the most-changed files
- If the PR is a simple docs/CI/test-only change, do a quick general review instead of full expert routing
